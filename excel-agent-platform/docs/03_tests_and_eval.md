# 03 Tests And Eval

## Main Evidence Pack

Run:

```bash
make eval
```

This runs backend lint plus unit, contract, integration, e2e, routing, and
performance checks. The current suite contains 51 tests.

Coverage includes:

- workbook profiling, reading, previewing, and writing
- formula DSL validation and execution
- haversine distance
- task routing and planner validation
- controlled API error envelopes
- MCP gateway normalization
- source/search lookup helpers
- typed `FactResult` contracts
- capitals and mountains golden accuracy
- exact final-project 4-row mountains workbook
- universal formula, group-share, lookup, and hybrid tasks
- 1000-row performance checks

## Evaluation Targets

- Numeric values within +/-10%.
- All known rows enriched or explicitly reported as `NOT_FOUND`.
- No secrets printed in logs or reports.

## Live Checks

Run the Docker stack in live mode:

```bash
OFFLINE_DEMO_SEED_FIRST=false docker compose up --build -d
make docker-health
```

Confirmed live signals:

- `/health` returns `model_status: live`
- `/health` returns `data_mode: live_sources_first`
- `/health/mcp` returns `strict: true` and all expected MCP tools
- live OpenRouter resolver maps `suma` to `+`
- live Wikidata lookup resolves Denali height to `6190`
- UI screenshots show `WIKIDATA` source evidence

## Screenshot Evidence

The README screenshots are regenerated by:

```bash
PLAYWRIGHT_BROWSERS_PATH=/tmp/ms-playwright make screenshots
```

The screenshot smoke uploads `tests/fixtures/mountains_final_project.xlsx`,
approves the web-backed plan, waits for `Run completed`, verifies source
evidence, downloads the workbook, and captures Swagger docs.
