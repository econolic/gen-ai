# ADR-003: Safe DSL

## Status

Accepted

## Decision

Represent executable table work as a narrow DSL with Pydantic validation, AST checks, whitelisted math helpers, and `col("Column Name")` references.

Supported operation types are:

- `formula`
- `group_share`
- `lookup`
- `distance`

Formulas are validated before execution and may use whitelisted helpers such as
`add`, `sub`, `mul`, `div`, and `col("Column Name")`. The DSL is generated by
rule-first planning and can fall back to OpenRouter only when deterministic
planning is not confident enough.

## Consequences

Formula, group-share, lookup, distance, and hybrid operations can be generated
without allowing arbitrary Python execution. Invalid plans fail closed into a
clarification path rather than running unsafe expressions.
